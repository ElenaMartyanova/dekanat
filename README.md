# Виртуальный деканат для иностранных студентов

Мультиязычный веб-сервис для информационной поддержки иностранных студентов.
Проект разработан в рамках выпускной квалификационной работы по направлению **38.03.05 «Бизнес-информатика»**.

Сервис помогает иностранным студентам быстро находить ответы на типовые учебные и административные вопросы, связанные с обучением, документами, визой, регистрацией, проживанием, личным кабинетом и другими университетскими сервисами.

## О проекте

**«Виртуальный деканат»** — это веб-приложение, которое объединяет типовые вопросы и ответы в одном интерфейсе и поддерживает работу на трёх языках:

* русский;
* английский;
* китайский.

Пользователь может выбрать язык, ввести вопрос, найти подходящий ответ, просмотреть похожие вопросы или сохранить новый вопрос, если готового ответа пока нет.

Администратор может управлять вопросами и ответами, обрабатывать новые обращения студентов, просматривать обратную связь и выгружать данные в Excel.

## Основные возможности

### Пользовательская часть

* выбор языка интерфейса;
* поиск ответа по пользовательскому запросу;
* просмотр тематических категорий;
* отображение основного ответа и похожих вопросов;
* сохранение необработанного вопроса;
* отправка обратной связи по желанию пользователя.

### Административная панель

* авторизация администратора;
* добавление, редактирование и удаление вопросов и ответов;
* управление мультиязычными записями;
* просмотр и обработка необработанных запросов;
* просмотр обратной связи;
* выгрузка данных в Excel.

### REST API

* получение списка категорий;
* поиск по вопросам и ответам;
* сохранение необработанных запросов;
* отправка обратной связи;
* получение ответов в формате JSON;
* проверка API через Swagger-документацию.

## Технологии

Проект реализован с использованием следующих технологий:

* **Python** — основной язык разработки;
* **FastAPI** — серверная часть приложения;
* **Jinja2** — HTML-шаблоны;
* **PostgreSQL** — база данных;
* **psycopg2** — подключение к PostgreSQL;
* **RapidFuzz** — нечёткое сравнение строк для поиска похожих вопросов;
* **OpenPyXL** — выгрузка данных в Excel;
* **HTML / CSS** — пользовательский интерфейс;
* **Swagger** — проверка REST API.

## Архитектура проекта

Сервис построен как клиент-серверное веб-приложение.

Пользователь и администратор работают с веб-интерфейсом через браузер.
Серверная часть на FastAPI обрабатывает запросы, выполняет поиск, обращается к базе данных PostgreSQL и возвращает результат пользователю.

База данных хранит:

* вопросы и ответы;
* языковые версии записей;
* тематические категории;
* необработанные запросы студентов;
* сообщения обратной связи;
* данные администратора.

## Механизм поиска

Поиск выполняется с учётом выбранного языка интерфейса.

Основные этапы обработки запроса:

1. Пользователь выбирает язык и вводит вопрос.
2. Серверная часть получает запрос.
3. Текст нормализуется: приводится к нижнему регистру и очищается от лишних символов.
4. Из базы данных выбираются записи на выбранном языке.
5. Запрос сравнивается с вопросами, ответами и ключевыми словами.
6. Для сравнения используется библиотека RapidFuzz.
7. Найденные записи сортируются по степени совпадения.
8. Пользователь получает основной ответ и похожие вопросы.
9. Если ответ не найден, вопрос можно сохранить как необработанный.

## Мультиязычная поддержка

Мультиязычность реализована на уровне:

* интерфейса;
* категорий;
* вопросов и ответов;
* результатов поиска.

При выборе языка сервис отображает данные только на выбранном языке.
Если язык введённого вопроса не совпадает с языком интерфейса, пользователь получает предупреждение.

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/your-username/virtual-dean-fastapi.git
cd virtual-dean-fastapi
```

### 2. Создать виртуальное окружение

```bash
python -m venv venv
```

Для Windows:

```bash
venv\Scripts\activate
```

Для macOS / Linux:

```bash
source venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Создать файл `.env`

Создайте файл `.env` в корне проекта и заполните параметры подключения к базе данных:

```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=virtual_dean
DB_USER=postgres
DB_PASSWORD=your_password

ADMIN_USERNAME=admin
ADMIN_PASSWORD=your_password
```

### 5. Подготовить базу данных PostgreSQL

Создайте базу данных PostgreSQL:

```sql
CREATE DATABASE virtual_dean;
```

Затем выполните SQL-скрипты проекта для создания таблиц и начального заполнения данных, если они предусмотрены в репозитории.

### 6. Запустить приложение

```bash
uvicorn main:app --reload
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000
```

Swagger-документация API:

```text
http://127.0.0.1:8000/docs
```
